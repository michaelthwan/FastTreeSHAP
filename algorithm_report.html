03The three optimizations in this fork's v2

All three attack constant factors in the hot loop — the asymptotics were already optimal. Each was gated on producing bit-identical output before being kept. Two further ideas (an iterative stack traversal, allocation hoisting) measured slower or within noise and were rejected; only measured wins were committed.

Incremental subset index

O(D) PER LEAF → O(1) · ≈ +11%

v2 looks up precomputed sums by a bitmask of "which path features passed". The original rescanned the whole path at every leaf; the index is now carried down the recursion and patched with one bit-op per step.

- unsigned ind = 0;                    // at every leaf
- for (i = 1; i <= depth; ++i)
-     if (path[i].one_fraction != 0)
-         ind += 1 << (i - 1);
+ // maintained during descent:
+ if (one_fraction != 0)
+     ind += 1u << (depth - 1);
+ // duplicate removed → bit splice:
+ ind = (ind & low) | (ind >> p) << (p-1);

Backtracking path

NO O(D) COPY PER NODE · ≈ +10%

Every node used to copy its parent's path array before touching it. Only one mutation ever hits the shared prefix — the duplicated-split removal — and it's exactly invertible, so the copy became mutate-and-undo.

- path = parent_path + depth;
- std::copy(parent_path,
-           parent_path + depth, path);
+ // shared array, in place
+ saved = path[dup_index];
+ /* ... recurse children ... */
+ shift_up(path, dup_index, depth);
+ path[dup_index] = saved;

Context-struct recursion

21 ARGS → 8 · ≈ +12%

The recursive call carried 14 loop-invariant pointers through every frame. They now live in one struct passed by reference — smaller frames, fewer spills, in a function called once per node per sample per tree.

- tree_shap_recursive_v2(num_outputs,
-   children_left, children_right, ...,
-   /* 21 arguments */);
+ struct TreeShapV2Context { /*14 fields*/ };
+ tree_shap_recursive_v2(ctx, node,
+   depth, residual, zero_f, one_f,
+   feature, ind);   // 8 args

04v3 — batched descent: changing the execution model itself

v0, v1 and v2 all keep one structural decision: the sample is the outer loop, and the tree is re-walked once per sample. Their improvements only changed the math at the leaves. v3 exploits a fact none of them use: along any root-to-leaf path, the path structure — features, zero-fractions, duplicate removals — is sample-independent. The only per-sample state is one pass/fail bit per split and a running residual.

Walk each tree once, carry all samples as vectors

v2 OPT 7.48s → v3 2.02s · ≈3.7× ON TOP · SAME OUTPUT

One recursion per tree instead of one per sample × tree. At each level, three arrays over the 2,000 samples replace per-sample recursion state: K (subset-index bits), R (residual products), B (branch bits from comparing a contiguous feature column against the threshold). Every hot operation becomes a dense, branch-light loop over samples — SIMD- and cache-friendly — and the traversal bookkeeping (path copies, duplicate removal, recursion overhead) is paid once per tree instead of 2,000 times. Same O(T·L·2^D + M·T·L·D) asymptotics; the win is all constant factor.

The inner loop, per tree level

scalar recursion (per sample) becomes:

// branch bits: one comparison per sample
for (m = 0; m < M; ++m)
    B[m] = x_col[m] <= threshold;
// extend K and R for the whole batch
for (m = 0; m < M; ++m) {
    K[m] = K_par[m] | (B[m] << bit);
    R[m] = B[m] ? R_par[m]
                : R_par[m] * zero_frac;
}
// leaf: φ update for all samples at once
// (identical arithmetic, identical order
//  → bit-identical output)

A structural bonus: because siblings rebuild their level's vectors from the parent's level, the duplicate-removal undo problem disappears — v3 needs no backtracking. And with per-thread transposed φ buffers, parallelism goes over trees with a single merge at the end. Validated like everything else: bit-identical on binary, 5-class multiclass, depth-14 regression, and gradient boosting, single- and multi-threaded.

06Why "same SHAP behaviour" is guaranteed

Correctness gate: every experiment was rejected unless its output matched a frozen reference (native shap's values) within atol 1e-8 — measured differences never exceeded ~9×10⁻¹⁵, i.e. floating-point noise, because the optimizations reorder bookkeeping, never arithmetic. Validated on binary classification (2,000 real samples), 5-class multiclass, depth-14 regression, and gradient boosting, with SHAP's own additivity check enabled. v1/v2 are exact algorithms from the Fast TreeSHAP paper (Yang, 2021), not approximations — and v3 applies the identical per-sample arithmetic in the identical DFS order as v2, just batched, so its outputs match v2's exactly (diff 0 between v3 and optimized v2 in single-thread runs). Every generation on this page computes the same Shapley values.
